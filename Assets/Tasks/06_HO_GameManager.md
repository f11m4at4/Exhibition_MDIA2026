# Task 06. `HO_GameManager`

## 목적
- 전시 시작 상태, 현재 관람 상태, 공통 참조를 중앙에서 정리한다.
- 다른 클래스가 직접 전역 상태를 바꾸지 않도록 마지막 통합 지점을 만든다.

## 구현 대상
- 클래스명/파일명: `HO_GameManager`
- 타입: `MonoBehaviour`
- 클래스 상단에 "전시 프로토타입의 전역 상태와 공통 참조를 관리한다" 주석을 작성한다.
- 초기화 함수에 어떤 순서로 참조를 검증하는지 주석을 작성한다.

## 선행 조건
- Task 01~05의 각 클래스가 개별 동작 가능 상태다.

## 이번 Task의 결과물
- `HO_GameManager.cs`
- 공통 참조 연결용 씬 매니저 오브젝트

## 구현 범위
- 전시 시작 상태 초기화
- 현재 관람 상태 저장
- 공통 참조 보관
- 잘못된 참조 누락 시 초기 로그 출력

## 권장 직렬화 필드
- `HO_PlayerController`
- `HO_UIManager`
- `HO_CuratorPresenter`
- `HO_ExhibitionFlowController`
- 프롤로그 시작 여부 같은 초기 설정값

## 권장 공개 메서드
- 전시 상태 변경 메서드 1개
- 현재 상태 조회 메서드 1개
- 해설 또는 UI 열림 여부 확인 메서드 1개

## 구현 단계
1. `HO_GameManager.cs`를 만들고 전역 상태 책임을 주석으로 명시한다.
2. `HO_PlayerController`, `HO_UIManager`, `HO_CuratorPresenter`, `HO_ExhibitionFlowController` 참조를 직렬화 필드로 받는다.
3. 전시 상태 열거형 또는 최소 상태값을 만든다. 예: `Prologue`, `Exploring`, `Presenting`, `Ending`.
4. 상태 변경 메서드는 `HO_GameManager`에서만 수행되게 만들고, 다른 클래스는 요청만 하도록 연결한다.
5. 시작 시 필수 참조 누락 여부를 검사하고 누락 시 경고 로그를 남긴다.
6. 시작 가능하면 공통 UI 초기화와 플레이어 조작 상태를 함께 정리한다.
7. UI 열림 상태와 플레이어 조작 잠금 상태를 함께 묶는 공통 제어 메서드를 만든다.

## 씬/인스펙터 작업
1. `GameManager` 빈 오브젝트를 만들고 `HO_GameManager`를 붙인다.
2. 이전 Task에서 만든 각 매니저/컨트롤러 참조를 모두 연결한다.
3. 플레이어, UI, 큐레이터, 흐름 컨트롤러의 상호 참조가 모두 맞는지 한 번 더 확인한다.
4. 씬 시작 시 상태를 `Prologue` 또는 팀이 정한 초기 상태로 둔다.

## 완료 기준
- 전역 상태 변경 지점이 `HO_GameManager`로 모인다.
- 참조 누락 시 조용히 실패하지 않고 로그를 남긴다.
- UI 열림/닫힘에 따라 플레이어 조작 잠금이 함께 맞물린다.
