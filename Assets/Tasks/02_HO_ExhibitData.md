# Task 02. `HO_ExhibitData`

## 목적
- 작품 감상과 큐레이터 해설에 필요한 최소 데이터를 `ScriptableObject`로 정리한다.
- 씬 오브젝트마다 별도 상호작용 스크립트를 붙이지 않고 데이터 자산 연결만으로 작품 정보를 관리할 수 있게 만든다.

## 구현 대상
- 클래스명/파일명: `HO_ExhibitData`
- 타입: `ScriptableObject`
- 클래스 상단에 "작품 감상에 필요한 메타데이터와 해설 문구를 보관한다" 주석을 작성한다.
- 데이터 묶음이 어떤 UI 단계에서 쓰이는지 필드 그룹 앞 주석을 작성한다.

## 선행 조건
- Task 01의 플레이어 이동이 완료되어 씬 탐색이 가능하다.

## 이번 Task의 결과물
- `HO_ExhibitData.cs`
- 작품별 `HO_ExhibitData` 에셋 샘플

## 생성 방식
- `CreateAssetMenu`를 사용해 생성 메뉴를 노출한다.
- 저장 위치는 `Assets/Data/Exhibits` 또는 팀이 쓰는 데이터 보관 경로로 통일한다.

## 구현 범위
- `작품 ID`
- `작품명`
- `작가명`
- `제작연도`
- `매체`
- `방 ID` 또는 방 구분용 값
- `대표 키워드`
- `1차 해설`
- `심화 정보`
- 필요 시 썸네일 같은 연결 리소스

## 공개 접근 원칙
- 읽기 전용 프로퍼티 또는 getter만 제공한다.
- 런타임 값을 바꾸는 setter는 만들지 않는다.

## 구현 단계
1. `CreateAssetMenu`를 포함한 `HO_ExhibitData.cs`를 만든다.
2. 모든 필드는 인스펙터에서 바로 확인 가능한 `[SerializeField] private` 또는 공개 읽기 프로퍼티 구조로 정리한다.
3. 1차 해설과 심화 정보는 서로 다른 필드로 분리한다.
4. 빈 문자열 또는 누락 데이터가 있을 때 이후 시스템이 로그를 남기고 패널을 열지 않도록, 데이터 유효성 확인에 필요한 기본 접근 메서드를 준비한다.
5. 대표 키워드는 배열 또는 리스트로 두되, 인스펙터 검수가 쉬운 형태로 유지한다.
6. `Assets/Resources` 또는 팀이 사용하는 데이터 보관 위치에 샘플 에셋을 몇 개 만든다.

## 씬/인스펙터 작업
1. 대표 작품 오브젝트를 정한다.
2. 각 작품 오브젝트 또는 자식 트리거 오브젝트에 나중에 연결할 데이터 에셋을 먼저 만든다.
3. 데이터 이름은 씬에서 찾기 쉬운 형태로 통일한다.
4. 작품마다 `HO_ExhibitData` 1개만 연결하는 규칙을 유지한다.

## 완료 기준
- 작품별 데이터 에셋을 인스펙터에서 확인할 수 있다.
- 1차 해설과 심화 정보가 분리돼 있다.
- 다음 Task에서 플레이어 감지 대상에 데이터를 연결할 준비가 되어 있다.
