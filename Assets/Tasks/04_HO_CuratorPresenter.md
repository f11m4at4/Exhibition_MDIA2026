# Task 04. `HO_CuratorPresenter`

## 목적
- 작품 해설 1단계와 심화 정보 2단계를 큐레이터 해설 흐름으로 보여준다.
- 플레이어와 2미터 떨어진 위치에서 설명하는 연출과 UI 표시를 한 클래스에서 처리한다.

## 구현 대상
- 클래스명/파일명: `HO_CuratorPresenter`
- 타입: `MonoBehaviour`
- 클래스 상단에 "큐레이터 해설 단계와 표시 연출을 제어한다" 주석을 작성한다.
- 단계 전환 함수에 1차 해설, 심화 정보, 종료 처리 역할 주석을 작성한다.

## 선행 조건
- Task 02의 `HO_ExhibitData`가 준비돼 있다.
- Task 03의 `HO_UIManager`가 준비돼 있다.

- 종료 호출 시 해설 UI 닫기

- 큐레이터 비주얼 루트
- 해설 표시 거리
- 다음 단계/종료 힌트 문구

- 다음 단계 진행 메서드 1개
- 종료 메서드 1개
- 현재 해설 중 여부 확인 메서드 1개

## 구현 단계
1. `HO_CuratorPresenter.cs`를 만들고 클래스/주요 함수 주석을 작성한다.
2. 필수 참조 누락 시 로그를 남기고 함수를 종료한다.
3. `HO_UIManager` 참조, 플레이어 트랜스폼 참조, 큐레이터 비주얼 또는 더미 오브젝트 참조를 직렬화 필드로 둔다.
4. 현재 해설 대상 데이터와 현재 단계(없음/1차 해설/심화 정보)를 저장한다.
5. 첫 호출 시 1차 해설, 두 번째 호출 시 심화 정보, 종료 호출 시 닫기 흐름을 메서드로 분리한다.
6. 플레이어의 전방 기준 2미터 떨어진 위치 계산 메서드를 만든다.
7. 데이터가 비어 있으면 UI를 열지 않고 `Debug.LogWarning`을 남긴다.

## 씬/인스펙터 작업
1. `CuratorPresenter` 오브젝트를 만들고 `HO_CuratorPresenter`를 붙인다.
2. 필요 시 큐레이터용 더미 오브젝트를 자식으로 배치하고 기본적으로 비활성 또는 대기 위치에 둔다.
3. `HO_UIManager`, 플레이어 트랜스폼 또는 플레이어 컨트롤러, 큐레이터 오브젝트 참조를 연결한다.

## 완료 기준
- 같은 작품에서 `E` 호출이 2단계 해설 흐름으로 이어진다.
- 해설 대상이 바뀌면 이전 텍스트가 남지 않는다.
- 큐레이터 위치가 플레이어 기준 2미터 앞 또는 측면 규칙으로 갱신된다.
