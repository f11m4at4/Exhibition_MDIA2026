# Task 03. `HO_UIManager`

## 목적
- 전시 프로토타입에서 공통으로 쓰는 최소 UI를 관리한다.
- 상호작용 안내, 해설 패널, 종료 메시지를 한 클래스에서 제어한다.

## 구현 대상
- 클래스명/파일명: `HO_UIManager`
- 타입: `MonoBehaviour`
- 클래스 상단에 "전시용 공통 UI 패널을 열고 닫는다" 주석을 작성한다.
- 주요 함수에 어떤 패널을 제어하는지 주석을 작성한다.

## 선행 조건
- Task 01의 플레이어 조작 구조가 준비돼 있다.
- Task 02의 작품 데이터 구조가 확정돼 있다.

## 이번 Task의 결과물
- `HO_UIManager.cs`
- 안내 프롬프트 UI
- 해설 패널 UI
- 종료 메시지 UI

## 구현 범위
- 한 번에 하나의 주요 패널만 열리게 한다.
- `E` 상호작용 안내 문구를 켜고 끈다.
- 큐레이터 해설용 텍스트 박스를 열고 닫는다.
- 에필로그 종료 메시지를 표시한다.

## 권장 직렬화 필드
- 프롬프트 패널/텍스트
- 해설 패널/제목/본문/힌트 텍스트
- 종료 메시지 패널/본문 텍스트

## 권장 공개 메서드
- `ShowPrompt` 또는 동등한 프롬프트 표시 메서드
- `HidePrompt`
- `ShowNarration` 또는 동등한 해설 표시 메서드
- `HideNarration`
- `ShowEndingMessage`
- `HideAllPanels`
- `HasOpenBlockingPanel`

## 구현 단계
1. `HO_UIManager.cs`를 만들고 역할 설명 주석을 작성한다.
2. 필수 참조 누락을 검사하고 누락 시 로그를 남긴다.
3. 프롬프트 텍스트, 해설 제목/본문/힌트 텍스트, 종료 메시지 텍스트를 참조하는 필드를 만든다.
4. 주요 패널을 제어하는 공개 메서드를 만든다.
5. 주요 패널이 열리면 다른 주요 패널을 닫는 보호 로직을 넣는다.
6. 초기 진입 시 모든 주요 패널이 꺼진 상태가 되도록 `Awake` 또는 `Start`에서 정리한다.

## 씬/인스펙터 작업
1. `Canvas_ExhibitionUI` 또는 동등한 캔버스를 만들고 안내 프롬프트, 해설 패널, 종료 메시지 영역을 배치한다.
2. 하위에는 `PromptPanel`, `NarrationPanel`, `EndingPanel` 3종만 우선 만든다.
3. `UIManager` 오브젝트를 만들고 `HO_UIManager`를 붙인다.
4. 텍스트 컴포넌트와 패널 오브젝트를 인스펙터에서 모두 연결한다.
5. 해설 패널은 측면 패널 또는 하단 텍스트 박스 한 종류로 통일한다.

## 완료 기준
- 프롬프트, 해설 패널, 종료 메시지를 코드 호출로 각각 제어할 수 있다.
- 해설 패널이 열릴 때 다른 주요 패널이 겹치지 않는다.
- 다음 Task에서 `HO_CuratorPresenter`가 바로 호출할 수 있다.

- `HO_PlayerController`가 프롬프트만 독립적으로 켜고 끌 수 있게 프롬프트 메서드를 분리해 둔다.
