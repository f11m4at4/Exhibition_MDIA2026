# Task 05. `HO_ExhibitionFlowController`

## 목적
- 프롤로그, 방 소개 포인트, 에필로그 종료까지의 전시 흐름을 관리한다.
- 대표 포인트 진입 여부와 대표 작품 감상 여부를 기준으로 단순한 진행 상태를 유지한다.

## 구현 대상
- 클래스명/파일명: `HO_ExhibitionFlowController`
- 타입: `MonoBehaviour`
- 클래스 상단에 "전시 구간 진행 상태와 방 소개 포인트를 관리한다" 주석을 작성한다.
- 상태 변경 함수에 어떤 구간을 확정하는지 주석을 작성한다.

## 선행 조건
- Task 01의 플레이어 이동이 완료돼 있다.
- Task 03의 UI, Task 04의 큐레이터 해설 호출 구조가 준비돼 있다.

## 이번 Task의 결과물
- `HO_ExhibitionFlowController.cs`
- 프롤로그/방 소개/에필로그용 트리거 배치 기준

## 구현 범위
- 프롤로그 시작 상태
- 방 소개 포인트 상태
- 대표 포인트 완료 처리
- 에필로그 종료 메시지 우선 출력

## 권장 직렬화 필드
- `HO_UIManager`
- `HO_CuratorPresenter`
- 프롤로그 메시지
- 종료 메시지
- 방 순서 목록

## 권장 공개 메서드
- 구간 트리거 처리 메서드 1개
- 대표 포인트 확인 메서드 1개
- 다음 방 진입 가능 여부 메서드 1개
- 전시 종료 여부 메서드 1개

## 구현 단계
1. `HO_ExhibitionFlowController.cs`를 만들고 상태 책임을 주석으로 정리한다.
2. 현재 방 인덱스, 완료 여부, 종료 여부를 저장하는 직렬화 필드 또는 내부 상태를 만든다.
3. 프롤로그 트리거 진입 시 첫 안내를 발생시키는 메서드를 만든다.
4. 방 소개 포인트 트리거 진입 시 중복 없이 한 번만 처리되게 한다.
5. 대표 작품 감상 완료 또는 지정 트리거 진입으로 방 완료를 판정하는 메서드를 만든다.
6. 에필로그 트리거 진입 시 종료 메시지를 한 번만 요청한다.

## 씬/인스펙터 작업
1. `FlowController` 또는 동등한 오브젝트를 만들고 스크립트를 붙인다.
2. 각 방 입구와 프롤로그 홀, 에필로그 라운지에 트리거 오브젝트를 만든다.
3. 트리거 이름은 `Trigger_PrologueEntry`, `Trigger_RoomA_Intro`, `Trigger_RoomA_Representative`, `Trigger_EpilogueEntry` 같은 식으로 통일한다.
4. 플레이어 태그 기준으로 트리거가 동작하도록 충돌 설정을 맞춘다.
5. 대표 포인트와 방 순서를 인스펙터에서 확인할 수 있게 목록 또는 배열로 연결한다.

## 완료 기준
- 프롤로그 -> 방 소개 -> 에필로그 흐름이 역순 없이 유지된다.
- 같은 트리거에 다시 들어가도 완료 처리가 중복되지 않는다.
- 에필로그 진입 시 종료 메시지가 우선 표시된다.
