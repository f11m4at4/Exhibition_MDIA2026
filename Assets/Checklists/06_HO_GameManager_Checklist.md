# Checklist 06. `HO_GameManager`

## 문서/정적 확인
- [ ] task 문서에 전역 상태 변경을 `HO_GameManager` 한 곳에서만 수행한다고 적혀 있다.
- [ ] 시작 시 공통 참조 누락 검사 절차가 문서에 포함돼 있다.
- [ ] 최종 인스펙터 연결 순서가 문서에 포함돼 있다.
- [ ] `단계별수행절차.md`에 최종 공통 참조 연결과 마무리 점검 순서가 반영돼 있다.

## 코드/구성 확인
- [ ] `HO_GameManager.cs` 파일명과 클래스명이 일치한다.
- [ ] 클래스와 주요 함수에 역할 설명 주석이 있다.
- [ ] 전역 상태 변경 메서드가 `HO_GameManager` 안에만 있다.
- [ ] 플레이어 조작 잠금과 UI 상태를 함께 제어하는 메서드가 있다.
- [ ] 참조 누락 시 경고 로그를 남기는 초기 검사 로직이 있다.

## 씬/인스펙터 확인
- [ ] `HO_PlayerController`, `HO_UIManager`, `HO_CuratorPresenter`, `HO_ExhibitionFlowController` 참조가 모두 연결돼 있다.
- [ ] 초기 상태가 프롤로그 또는 팀이 정한 시작 상태로 설정돼 있다.

## Play Mode 검증
- [ ] 미검증: 해설 UI가 열리면 플레이어 조작이 잠긴다.
- [ ] 미검증: 해설 UI가 닫히면 플레이어 조작이 다시 풀린다.
- [ ] 미검증: 종료 상태에서는 추가 안내보다 종료 메시지가 우선된다.
