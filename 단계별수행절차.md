# 단계별수행절차

## 목적
- `Assets/Tasks` 폴더의 task 문서를 실제 구현 순서대로 실행할 수 있게 정리한다.
- 각 단계는 "task 문서 실행", "Unity 실습", "의존성 보류", "checklist 반영" 순서로 마무리한다.

## 공통 작업 원칙
1. 각 task 문서는 반드시 서브에이전트로 실행한다.
2. 현재 단계에서 요구하지 않는 의존성 클래스는 미리 구현하지 않는다.
3. Unity 실습은 `Exhibition` 단일 씬에서만 진행한다.
4. 입력은 Old Input System만 사용한다.
5. 한 단계가 끝나면 대응 checklist 문서에 정적 확인 항목과 미검증 항목을 함께 점검한다.
6. 작업할 Scene 파일은 `Assets/Scenes/Exhibition.unity`이다.

## 1단계. `01_HO_PlayerController.md`
1. `Assets/Tasks/01_HO_PlayerController.md`를 서브에이전트로 실행한다.
2. `HO_PlayerController.cs`만 구현하고, `HO_UIManager`, `HO_CuratorPresenter`는 아직 구현하지 않는다.
3. Unity에서 `Exhibition` 씬을 연다.
4. `Capsule` 게임오브젝트를 만들고 이름을 `Player`로 변경한다.
5. `Player`에 `CharacterController`를 추가한다.
6. `Player` 자식으로 `CameraRoot`를 만들고, 그 아래에 `Main Camera`를 배치한다.
7. `Player`에 `HO_PlayerController.cs`를 컴포넌트로 할당한다.
8. 카메라 참조, 이동 속도, 감도, 중력, 회전 제한값을 인스펙터에 입력한다.
9. 상호작용용 레이어 마스크는 비워 두거나 작품 트리거 레이어만 임시 지정한다.
10. [01_HO_PlayerController_Checklist.md](D:/unity/Exhibition/Assets/Checklists/01_HO_PlayerController_Checklist.md)를 열고 문서/구성 항목을 확인한다.

## 2단계. `02_HO_ExhibitData.md`
1. `Assets/Tasks/02_HO_ExhibitData.md`를 서브에이전트로 실행한다.
2. `HO_ExhibitData.cs`만 구현하고, 작품 상호작용용 별도 MonoBehaviour는 만들지 않는다.
3. `Assets/Data/Exhibits` 또는 팀 데이터 경로를 만든다.
4. 대표 작품 수만큼 `HO_ExhibitData` 에셋을 생성한다.
5. 각 에셋에 작품명, 작가명, 제작연도, 매체, 키워드, 1차 해설, 심화 정보를 입력한다.
6. 씬의 작품 오브젝트 이름과 데이터 에셋 이름을 대응되게 정리한다.
7. 아직 `HO_PlayerController`와 실제 상호작용 연결은 하지 않는다.
8. [02_HO_ExhibitData_Checklist.md](D:/unity/Exhibition/Assets/Checklists/02_HO_ExhibitData_Checklist.md)를 열고 데이터 필드와 에셋 준비 여부를 반영한다.

## 3단계. `03_HO_UIManager.md`
1. `Assets/Tasks/03_HO_UIManager.md`를 서브에이전트로 실행한다.
2. `HO_UIManager.cs`만 구현하고, 큐레이터 연출 로직은 넣지 않는다.
3. Unity에서 `Canvas_ExhibitionUI`를 만든다.
4. 하위에 `PromptPanel`, `NarrationPanel`, `EndingPanel` 3개만 만든다.
5. 필요한 텍스트 오브젝트를 각 패널 하위에 배치한다.
6. 빈 오브젝트 또는 Canvas 루트에 `HO_UIManager.cs`를 붙인다.
7. 패널과 텍스트 참조를 인스펙터에 연결한다.
8. 씬 시작 시 패널이 모두 닫히도록 초기 상태를 확인한다.
9. 아직 `HO_CuratorPresenter`의 실제 호출은 연결하지 않는다.
10. [03_HO_UIManager_Checklist.md](D:/unity/Exhibition/Assets/Checklists/03_HO_UIManager_Checklist.md)를 열고 UI 구성 항목을 반영한다.

## 4단계. `04_HO_CuratorPresenter.md`
1. `Assets/Tasks/04_HO_CuratorPresenter.md`를 서브에이전트로 실행한다.
2. `HO_CuratorPresenter.cs`를 구현하고, 해설 단계 전환과 2미터 위치 계산만 우선 만든다.
3. Unity에서 `CuratorPresenter` 오브젝트를 만든다.
4. 필요하면 큐레이터용 더미 오브젝트를 자식으로 만든다.
5. `CuratorPresenter`에 `HO_CuratorPresenter.cs`를 붙인다.
6. `HO_UIManager`, `Player`, 큐레이터 비주얼 참조를 인스펙터에 연결한다.
7. 작품 해설 1차/심화 정보 표시 메서드만 연결하고, 방 흐름 제어는 아직 넣지 않는다.
8. [02_HO_ExhibitData.md](D:/unity/Exhibition/Assets/Tasks/02_HO_ExhibitData.md)의 데이터 구조와 맞는지 다시 확인한다.
9. [04_HO_CuratorPresenter_Checklist.md](D:/unity/Exhibition/Assets/Checklists/04_HO_CuratorPresenter_Checklist.md)를 열고 문서/구성 항목을 반영한다.

## 5단계. `05_HO_ExhibitionFlowController.md`
1. `Assets/Tasks/05_HO_ExhibitionFlowController.md`를 서브에이전트로 실행한다.
2. `HO_ExhibitionFlowController.cs`를 구현하고, 방 상태와 트리거 중복 방지 로직만 우선 만든다.
3. Unity에서 `FlowController` 오브젝트를 만든다.
4. 프롤로그 홀, 각 방 입구, 대표 포인트, 에필로그 라운지에 트리거 오브젝트를 배치한다.
5. 트리거 이름을 `Trigger_PrologueEntry`, `Trigger_RoomA_Intro`, `Trigger_RoomA_Representative`, `Trigger_EpilogueEntry` 규칙으로 맞춘다.
6. `HO_ExhibitionFlowController.cs`를 `FlowController`에 붙인다.
7. 방 순서 목록과 메시지 필드를 인스펙터에 연결한다.
8. 대표 작품 감상 완료와 연결되는 호출 지점만 정하고, 최종 상태 통합은 아직 `HO_GameManager`에서 하지 않는다.
9. [05_HO_ExhibitionFlowController_Checklist.md](D:/unity/Exhibition/Assets/Checklists/05_HO_ExhibitionFlowController_Checklist.md)를 열고 트리거/상태 항목을 반영한다.

## 6단계. `06_HO_GameManager.md`
1. `Assets/Tasks/06_HO_GameManager.md`를 서브에이전트로 실행한다.
2. 마지막으로 `HO_GameManager.cs`를 구현한다.
3. Unity에서 `GameManager` 빈 오브젝트를 만든다.
4. `GameManager`에 `HO_GameManager.cs`를 붙인다.
5. `HO_PlayerController`, `HO_UIManager`, `HO_CuratorPresenter`, `HO_ExhibitionFlowController` 참조를 모두 연결한다.
6. 초기 상태를 `Prologue` 또는 팀이 정한 시작 상태로 맞춘다.
7. 해설 UI 열림 시 플레이어 조작 잠금이 함께 동작하도록 연결 상태를 점검한다.
8. 참조 누락 시 로그가 남도록 초기화 순서를 확인한다.
9. [06_HO_GameManager_Checklist.md](D:/unity/Exhibition/Assets/Checklists/06_HO_GameManager_Checklist.md)를 열고 공통 참조/상태 항목을 반영한다.

## 마무리 점검 순서
1. `01_HO_PlayerController_Checklist.md`부터 `06_HO_GameManager_Checklist.md`까지 순서대로 다시 연다.
2. 문서/정적 확인과 코드/구성 확인을 먼저 체크한다.
3. Play Mode를 아직 실행하지 않았다면 모든 실행 검증 항목은 `미검증`으로 유지한다.
4. Play Mode 검증은 `이동` -> `시점 회전` -> `E 상호작용` -> `해설 2단계` -> `종료 메시지` 순서로 진행한다.
