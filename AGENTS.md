# AGENTS.md

## 범위
- 이 문서는 `D:\unity\Exhibition` Unity 프로젝트 작업 규칙이다.
- 참조 문서: `mds/1.초기화설정.md`, `Assets/Design/콘텐츠기획.md`
- 이 에이전트가 직접 소유하고 수정하는 파일은 `D:\unity\Exhibition\AGENTS.md`만 한다.
- 다른 작업자의 변경 사항은 되돌리거나 덮어쓰지 않는다.

## 작업 원칙
- 구현 범위는 항상 프로토타입 버전에 한정한다. 과한 일반화, 최적화, 대규모 리팩터링은 피한다.
- md 파일 실행은 서브에이전트를 통해 진행한다.
- `Assets/Task` 폴더에 있는 Task 실행 요청은 반드시 서브 에이전트를 통해 진행한다.
- 사용자 요청을 실행했으면 구현 결과를 체크리스트 문서에도 반드시 확인/반영한다.

## Unity 기준
- 입력 시스템은 New Input System이 아니라 Old Input system을 사용한다.
- 기본 조작 기준은 `WASD` 이동, `Mouse` 시점, `E` 상호작용을 우선한다.
- 새로 만드는 모든 클래스 이름은 반드시 `HO_` 접두사를 사용한다.
- Unity C# 코드는 파일명과 클래스명을 일치시키고, 하나의 파일에는 하나의 주 클래스만 둔다.
- 네이밍은 `PascalCase`(class/public member), `camelCase`(local/private field) 기준을 따른다.
- Serialized field가 필요하면 `private` 필드에 `[SerializeField]`를 우선 사용한다.
- 매직 넘버는 필요한 경우 상수 또는 직렬화 필드로 분리한다.
- 구현된 클래스와 함수에는 반드시 어떤 역할을 하는지 설명하는 **주석**을 단다.

## 구현 태도
- 전시 흐름, 상호작용, UI는 콘텐츠 기획 문서의 의도에서 벗어나지 않게 최소 구현으로 맞춘다.
- 변경 전에는 관련 파일과 기존 구조를 먼저 확인하고, 영향 범위를 작게 유지한다.
- 작업 결과 보고에는 변경 파일 경로와 짧은 요약을 함께 남긴다.
